### 3. The Problem (Pain Point Validation)

#### What Producers Struggle With
1. **DAWs offer basic swing only:** 50-66% triplet swing (too "bouncy")
2. **Dilla's sweet spot is 57-61%:** Quintuplet/septuplet feel (hard to achieve)
3. **Per-track timing is complex:** Different instruments need different offsets
4. **MPC groove templates require import:** Not native to most DAWs
5. **Manual nudging is tedious:** "Micro adjustments using 192 grid"

#### Current Solutions (All Inadequate)
| Solution | Problem |
|----------|---------|
| DAW built-in swing | Only basic triplet swing, global application |
| MPC groove templates | Requires import, not customizable live |
| Manual note nudging | Time-consuming, no real-time preview |
| Playing live unquantized | Requires drumming skill most don't have |
| Expensive hardware (MPC) | $1,000+ for authentic feel |

#### Quote from MPC Forums:
> "Turn off quantize and practice, practice, practice!"

This is the problem: **the current solution is "get better at drumming."** A software tool that automates this is a clear opportunity.

## Dilla Engine Technical Verification

### Comparison: Technical Spec vs Current Implementation

| Spec Requirement | Expected | Our Implementation | Status |
|-----------------|----------|-------------------|--------|
| **96 PPQN Quantization** | `round(t × 96) / 96` | `quantizeToMPCTicks()` | MATCH |
| **Swing Sweet Spot** | 57-61% | 57%-60% | MATCH |
| **Swing on even 16ths only** | Steps 2,4,6... | `stepIdx % 2 === 1` | MATCH |
| **Kick timing** | Late/Dragging (+offset) | +15ms to +45ms | MATCH |
| **Snare timing** | Early/Rushing (-offset) | -0 to -10ms | MATCH |
| **Micro-variation** | Human jitter | ±2-8ms Gaussian | MATCH |
| **Velocity modulation** | Dynamic accents | Hi-hat downbeat boost | MATCH |

### Design Decision: "Friction" Implementation

**The Technical Spec says:**
- Kick: Straight (50%) OR Late
- Hi-hats: Heavy Swing (62%)
- Friction = swung hats vs straight kick

**Our Implementation:**
- Kick: Swings hardest (1.0 multiplier) + late offset
- Hi-hats: Mostly rigid (0.3 multiplier)
- Friction = lazy kick vs rigid hats

**Both are valid interpretations.** Different Dilla tracks exhibit different friction patterns:
- "Donuts" era: Kick drags, hats provide steady pulse
- "Slum Village" era: Hats swing, kick anchors

Our implementation leans "Donuts" style where the kick is the drunk element.

### Core Algorithm Formula

```
Our timing formula:
t_final = t_grid + pushPull + tupletSwing + aiOffset + microVariation
        → quantizeToMPC(result)

This is equivalent to the spec's:
t_final = t_step + O_groovae + B_dilla
```

## Feature Roadmap

### Phase 1: Core Export & Presets (MVP for Launch)

**Goal:** Enable producers to actually USE their beats outside the browser

#### 1.1 WAV Export
- Use `OfflineAudioContext` to render full pattern (4 bars = 16 steps × 4 loops)
- Apply all humanization/ghost notes during offline render
- Output as 16-bit/44.1kHz WAV (PRO: 24-bit/48kHz option)

**Files to create/modify:**
- `src/core/AudioExporter.js` (new)
- `src/core/AudioEngine.js` (add offline render method)
- `index.html` (add export button)
- `src/main.css` (export modal styles)

#### 1.2 Preset Patterns

| Pattern | BPM | Style | Notes |
|---------|-----|-------|-------|
| Boom Bap Classic | 90 | 90s hip-hop | Kick on 1/3, snare 2/4, 8th hats |
| Trap 808 | 140 | Modern trap | Rolling hats, sparse kick, hard snare |
| Lo-Fi Chill | 85 | Lo-fi beats | Swing heavy, ghost snares |
| Drill UK | 140 | UK drill | Sliding 808, syncopated snare |
| Neo Soul | 95 | Soulful | Ghost-heavy, Dilla-style swing |
| Boom Bap Uptempo | 100 | Faster 90s | Amen-influenced |

**Files to create/modify:**
- `src/presets/patterns.js` (new)
- `src/components/PresetSelector.js` (new)
- `src/core/Store.js` (add LOAD_PRESET action)

#### 1.3 Pattern URL Sharing (PRO)
- Serialize state to JSON → base64 encode → URL hash
- Format: `https://app.com/#pattern=eyJzdGVwcy...`

**Files to create/modify:**

#### 1.4 Tap Tempo

**Files to create/modify:**
- `src/components/TapTempo.js` (new)

### Phase 2: Pro Export Features (Monetization Gate)

#### 2.1 MIDI Export
- Use `midi-writer-js` library
- Export pattern as Type 0 MIDI file
- Map tracks to GM drum notes: Kick=36, Snare=38, HH=42, Open=46
- Include humanization timing offsets and velocity

**Files to create/modify:**
- `src/core/MidiExporter.js` (new)
- `package.json` (add midi-writer-js)

#### 2.2 Stem Export
- Render each track separately using OfflineAudioContext
- Produce 4 WAV files: kick.wav, snare.wav, hihat.wav, openhat.wav
- Zip together using JSZip library

**Files to create/modify:**
- `src/core/AudioExporter.js` (add stemExport method)
- `package.json` (add jszip)

#### 2.3 Custom Sample Upload
- Use `<input type="file" accept="audio/*">`
- Load into AudioBuffer via `decodeAudioData`
- Store in IndexedDB for persistence

**Files to create/modify:**
- `src/core/SampleManager.js` (new)
- `src/core/AudioEngine.js` (support custom buffers)
- `src/components/SampleUploader.js` (new)
